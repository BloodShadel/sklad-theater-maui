# Склад театра

Приложение для учёта склада театра: **Android-клиент** и **сервер для Windows** с авторизацией по логину и паролю (JWT).

---

## Сервер (Windows)

- **Стек:** Node.js, Express, JSON-файл (хранилище), JWT, bcrypt
- **Порт:** 3000

### Запуск

1. Установите [Node.js](https://nodejs.org/) (LTS).
2. В папке `server` выполните:

```bash
cd server
npm install
npm start
```

Сервер будет доступен по адресу: `http://localhost:3000`

### Веб-панель администратора

После запуска сервера откройте в браузере: `http://localhost:3000/admin`

**Функции админ-панели:**
- Создание учётных записей сотрудников
- Сброс паролей
- Удаление сотрудников
- Назначение ролей (администратор/сотрудник)

**Первый запуск:** при отсутствии пользователей админ-панель предложит создать администратора.

### API

| Метод | URL | Описание |
|-------|-----|----------|
| POST | `/api/auth/login` | Вход (login, password) |
| GET | `/api/auth/me` | Текущий пользователь (заголовок `Authorization: Bearer <token>`) |
| GET | `/api/admin/users` | Список пользователей (только админ) |
| POST | `/api/admin/users` | Создать пользователя (только админ) |
| DELETE | `/api/admin/users/:id` | Удалить пользователя (только админ) |
| PUT | `/api/admin/users/:id/password` | Сбросить пароль (только админ) |
| GET | `/api/version` | Проверка версии приложения |
| GET | `/download/apk` | Скачать APK последней версии |

Данные пользователей сохраняются в `server/data/store.json` (создаётся при первом запуске).

## Android-приложение

- **Язык:** Kotlin
- **Минимальная SDK:** 24
- **Сборка:** Android Studio или Gradle

### Сборка и запуск

1. Установите [Android Studio](https://developer.android.com/studio).
2. Откройте папку `android` как проект.
3. Подключите устройство или запустите эмулятор.
4. Выполните **Run** (или `./gradlew installDebug` из папки `android`).

### Подключение к серверу

В приложении указан сервер: `http://80.251.125.152:3000/`. Убедитесь, что сервер запущен и на роутере/ПК открыт порт 3000 для входящих подключений.

### Сборка APK

1. Установите [Android Studio](https://developer.android.com/studio) (появится Android SDK).
2. В папке `android` создайте файл `local.properties` с путём к SDK, например:
   `sdk.dir=C\:\\Users\\ВАШ_ЛОГИН\\AppData\\Local\\Android\\Sdk`  
   Или скопируйте `local.properties.example` в `local.properties` и отредактируйте.
3. **Через скрипт:** запустите `android\build-apk.bat`. APK появится в корне проекта как `sklad-theater-debug.apk` и в `android\app\build\outputs\apk\debug\app-debug.apk`.
4. **Через Android Studio:** откройте папку `android`, Build → Build Bundle(s) / APK(s) → Build APK(s).

## Система авторизации

1. **Регистрация отключена в приложении.** Сотрудников создаёт администратор через веб-панель `/admin`.
2. **Вход:** логин и пароль; сервер возвращает JWT-токен.
3. Приложение сохраняет токен и при открытии главного экрана проверяет его через `/api/auth/me`; при невалидном токене открывается экран входа.
4. Выход — очистка токена и переход на экран входа.

---

## Система обновлений

1. При запуске приложение проверяет версию через `/api/version?version=X.Y.Z`.
2. Если версия на сервере новее — показывается диалог с предложением обновиться.
3. **Для обновления версии:**
   - Измените `APP_VERSION` в `server/src/index.js` (например, `'1.1.0'`)
   - Измените `versionName` в `android/app/build.gradle.kts` (например, `"1.1.0"`)
   - Соберите новый APK и замените `sklad-theater-debug.apk` в корне проекта
   - Перезапустите сервер
